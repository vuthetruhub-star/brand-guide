Brand Description
D1AGENCY is a digital agency with a technology-first visual language.
The brand preserves terminal / hacker DNA — Orbitron display, neon primary #00FF41, subtle scanline ambient, and cursor blink — but expresses it through modern, disciplined materiality.
It should feel closer to Linear, Vercel, and Resend in material quality, while still clearly reading as an agency building dev-tool-grade systems rather than a generic SaaS template.

Visual Direction
Aesthetic
Modern technical professional with terminal DNA. High-contrast, dark-first, engineering-rooted, and never hacker cosplay.

Mood
Confident, precise, calm, and deliberate. The brand carries DNA without becoming theatrical.

Era and Inspiration
Contemporary developer tools from 2024–2026, with disciplined hacker accents.

Design Goal
Preserve 60% of terminal DNA through type, color, and motion:

Type: Orbitron

Color: neon #00FF41

Motion: cursor blink, subtle scanline, terminal animation

Remove 40% of theatrics:

Heavy CRT ambient

Glitch ambient overload

Exo 2

Intel One Mono

Color System
Strategy
Restrained dark neutrals plus a signature neon green accent.

Primary neon #00FF41 should occupy only 5–10% of any surface, mainly for CTAs, status, key inline highlights, and the brand mark

Neutrals should occupy 80–90% of surfaces

Only 3 semantic colors are used: critical, notice, and info

No redundant orange warning state

Light mode uses #3DE07B as the primary brand color for decorative use, button backgrounds, and the brand mark

Light mode text and CTA contrast must use --primary-strong: #127A3E

Mono-hue green discipline must remain consistent across both modes

Elevation and Depth
Depth Principles
There are no traditional drop shadows. Depth comes from three techniques:

Inner-border highlight
box-shadow: inset 0 1px 0 rgba(255, 255, 255, 0.04) on the top edge of cards

Surface layering
Background → Surface → Surface Alt

Disciplined green glow
Local glow on active or focused interactive elements and status indicators

Glow Specifications
Max blur radius: 16px

Max opacity: 0.22 in both dark and light mode

Dark glow color: rgba(0, 255, 65, X)

Light glow color: rgba(61, 224, 123, X)

Glow Use Cases
Allowed:

Primary CTA

Focused input

Selected panel

Brand mark

Status indicators

Signature moments

Never:

Ambient full-page green lighting

Every card glowing by default

Multiple glow layers stacked everywhere

Motion System
Philosophy
Motion should be crisp and purposeful while preserving terminal rhythm. Cursor blink, scanline subtlety, and terminal-style reveal are core identity signals rather than decoration.

Motion Principles
Enter animations use ease-out

Terminal accent moments may use ease-snap

Exit animations should be faster than enter animations

Press / release asymmetry: 80ms press, 160ms release feel

Cursor blink remains visible in inputs and signature terminal blocks

Primary CTA hover may include subtle scanline flicker around 80ms

Status indicators pulse softly every 2s

Hover must be gated by @media (hover: hover) and (pointer: fine)

Reduced motion must disable blink, scan, glitch, and transforms while keeping opacity and color transitions

Frequent keyboard-driven UI actions should not animate excessively

Effect Rules
Do not stack more than 2 effects at once

Use only one dominant effect per scene whenever possible

Do not use effect stacks as decoration

Interaction States
Every interactive component must define all 6 states:

Default — base styling

Hover — slight brightness increase, border lift, or glow bump

Focus — 0 0 0 2px var(--primary) with 2px offset

Active / Pressed — scale(0.98) or color shift to --primary-pressed

Disabled — opacity: 0.4, cursor: not-allowed, no hover effect

Loading — skeleton or inline progress with scan animation, not an empty spinner

Accessibility
Core Rules
Minimum body text contrast must meet WCAG AA (4.5:1)

Important CTAs should aim for AAA when practical

In light mode, never use #3DE07B for text

Focus rings must always remain visible

Never remove outlines without a proper accessible replacement

All motion must respect prefers-reduced-motion: reduce

Hover effects must be gated by hover-capable fine pointers

Semantic HTML first, ARIA only when needed

Minimum touch target size: 44 × 44px

UI Principles
Core Principles
Dark mode is primary; design everything dark-first

Neon green #00FF41 is signature DNA and should remain an accent only

Mono plus Orbitron carry the technical DNA

One discipline at a time: one main effect per scene

Motion expresses identity, not decoration

Materiality is more important than decoration

Modular layouts are better than maximalist layouts

Readability is non-negotiable

Do
Use Orbitron for hero and display moments

Use Geist Sans for headings and body

Use Geist Mono for UI accents, labels, and code

Use #0A0B0A instead of pure black

Use #00FF41 as the dark primary accent

Use #3DE07B decoratively in light mode and #127A3E for readable text

Use inner-border highlights on cards

Use stepped radius values only

Use the 4px spacing scale only

Gate hover effects with media queries

Respect reduced-motion preferences

Use tabular numerals for data

Keep cursor blink and status pulse on by default

Pick only 2–4 optional motion enhancements per project

Don’t
Do not use #000000 or #FFFFFF

Do not use neon green for body text

Do not use Orbitron for paragraphs, captions, or UI labels

Do not exceed scanline opacity 0.025 for ambient use

Do not stack more than 2 effects together

Do not use drop shadows

Do not load more than 3 font families

Do not use pill radius outside status badges

Do not break the 4px spacing system

Do not use sharp corners on more than 1 element per page

Do not animate routine UI longer than 320ms, except for approved ambient cycles

Do not use more than 4 optional enhancements

Do not use #3DE07B as text in light mode
